# Oceana Project — Claude Operating Guide

## Project
Rebuilding the website for a luxury private yacht charter company in Southeast Alaska (M/Y Oceana, Juneau-based).

## Primary Goal
High-converting direct-response marketing site. Not a visual redesign — a conversion funnel built around: offer clarity, CTA placement, trust stacking, and booking flow.

## Core Business Context
- 8-day all-inclusive private yacht expeditions through the Inside Passage
- Up to 6 guests, full crew of four (Captain, Chef, First Mate, Stewardess)
- From $65,000 per charter — high-ticket, single-group, no strangers
- Original platform: Squarespace. Prototype is a static multi-page HTML/CSS site

## Working Priorities
1. Conversion performance over aesthetics
2. Clear offer structure
3. Strong CTA placement
4. Trust-building throughout the page
5. Fast iteration with minimal token waste

## File Reading Rules
Before doing work:
- Read this file first
- Read only the specific files needed for the current task
- Do not scan the entire project unless explicitly asked
- Prefer README/index files before opening detailed files

## Folder Usage Rules
- `research/` contains competitor and market research only
- `planning/` contains approved strategy and build decisions
- `site/` contains the actual prototype
- `deliverables/` contains client-facing notes and summaries

## Build Rules
- Do not rewrite project structure without approval
- Do not create duplicate strategy docs if one already exists
- Update existing files when appropriate instead of making new ones
- Keep outputs concrete and implementation-ready
- Prefer small edits over large refactors

## Output Style
- Be direct
- Be specific
- Avoid generic marketing language
- Think like a conversion strategist, not just a designer

## Current Phase
The build is in progress. Prototype covers: homepage, experience, itinerary, crew, gallery, about, FAQ, contact.

Current work: layout refinement, image integration, spacing polish, copy tightening, conversion iteration.

Phase order for reference: research → synthesis → planning → **build (active)** → revision/testing
